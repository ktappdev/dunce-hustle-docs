Thanks for helping improve the docs.

## Contribution Principles

- Keep mechanics aligned with source code behavior.
- Prefer explicit formulas and numeric ranges over vague wording.
- Preserve existing accurate content when expanding pages.
- Use clear section headers and short, scannable tables.

## Recommended Workflow

1. Fork and create a branch.
2. Update docs under `src/content/docs`.
3. Keep Starlight frontmatter (`title`, `description`) on every page.
4. Run local build checks.
5. Open PR with summary + changed files.

## Style Expectations

- Use single source of truth from backend constants/logic.
- Call out edge cases where behavior is non-obvious.
- Keep terminology consistent with glossary.
- Avoid speculative mechanics not present in code.

## Good PR Checklist

- [ ] Contradictions removed
- [ ] Formulas/percentages verified
- [ ] Navigation updated for new pages
- [ ] No dead links
- [ ] Build passes locally

## Reporting Documentation Issues

When filing issues, include:

- Page path and section title
- What seems incorrect
- Expected behavior (with code pointer if possible)
- Screenshot or reproduction context if relevant
