description: Centralized settings for modes, economy, combat, and events.

The game's mechanics, economy balance, and match settings are centralized in `convex/gameConfig.ts`. This allows rapid tuning without touching core logic.

## Configuration File
- **Path:** `convex/gameConfig.ts`
- **Purpose:** Controls hardcoded values for economy, combat, events, and game modes.

## Key Settings

### 1. Game Modes (`GAME_MODES`)
Defines the available game types for multiplayer lobbies. Each mode controls the pace of the game.

- `dayDurationSeconds`: How long a "day" lasts in real time (e.g., 60s for Blitz).
- `totalDays`: The fixed length of a match (default: 30 days).
- To add a new mode, add a new entry to the `GAME_MODES` object; the UI renders it automatically.

### 2. Economy (`ECONOMY`)
Controls market volatility and trading difficulty.

- `BUY_PRESSURE` / `SELL_PRESSURE`: How much prices shift when players trade. Higher values create a more volatile market.
- `CHEAP_MULTIPLIER` / `EXPENSIVE_MULTIPLIER`: Base discount (0.7x) or markup (1.4x) for goods in preferred locations.
- `ITEM_RESALE_VALUE`: Percent value retained when selling weapons/armor back to the shop (default: 50%).

### 3. Combat & Mugging (`COMBAT`)
Balances the risk/reward of PvP interaction.

- `ATTACK_ENERGY_COST`: Cost to initiate a mugging.
- `NEWBIE_PROTECTION_NET_WORTH`: Players below this net worth cannot be targeted ($2,000).
- `LOOT_CASH_MIN` / `LOOT_CASH_MAX`: Percent of target cash stolen on success (10–25%).
- `LOOT_GOODS_MIN` / `LOOT_GOODS_MAX`: Percent of target inventory stolen on success.
- `GOON_COST`: Price to hire defensive NPCs.

### 4. Events (`EVENTS`)
Controls the severity of random encounters during travel.

- `BUST_GOODS_LOSS`: % of inventory lost when caught by CANU (30%).
- `SCRAPE_CASH_LOSS`: % of cash lost when robbed by Scrape Head (20%).
- `TRAVEL_RISK_REDUCTION`: How much your "heat" decreases per travel action.

## How to Tune
1. Open `convex/gameConfig.ts`.
2. Modify constant values (e.g., change `STARTING_STATS.CASH` from 10000 to 15000).
3. Save the file.
4. Run `npx convex dev` to push changes. The backend and frontend will reflect the new balance.
